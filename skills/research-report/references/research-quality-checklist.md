# Research Quality Checklist

Use this file as the review pass before delivering a report.

## Content quality

- Answer the user's actual question, not a neighboring one.
- Lead with the decision-relevant conclusion.
- Separate observations, interpretation, and recommendations.
- Remove generic filler and repetitive restatement.
- Prefer concrete numbers, examples, and named entities.

## Source quality

- Prefer primary sources: official blogs, filings, docs, product pages, papers, company statements.
- Use secondary coverage only when it adds synthesis or points to a primary source.
- Avoid weak or citation-free SEO spam.
- Note when evidence is thin, stale, or contradictory.

## Structure quality

- Executive summary should stand alone.
- Findings should be grouped into meaningful themes.
- Risks should be real, not ceremonial.
- Recommendations should follow from findings.
- Sources should be inspectable and directly linked.

## Markdown quality

- Use headings hierarchically.
- Use bullets for scanability.
- Keep tables only when they add real compression of information.
- Ensure links render cleanly on GitHub.

## PDF readiness

- Ensure heading hierarchy is clean.
- Avoid broken long URLs in body text when possible.
- Keep sections from becoming one giant wall of text.
- Check the HTML preview before exporting to PDF.
